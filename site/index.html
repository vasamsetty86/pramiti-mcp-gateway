Why it exists

A single MCP server can put a delete_repository, a transfer_funds, and a harmless search in the same agent's reach. If the agent is prompt-injected, it will be talked into calling the wrong one. Prompt-level defenses don't help here — the danger isn't what the agent says, it's what it can do. Before you can gate agent actions, you have to see them. This is the "see them" step, and it's free.

What you get

Scan (offline)

Classify any MCP server's tools by access, reversibility, sensitive-data exposure, and arbitrary power. Zero dependencies. CI-gateable.

Scan (live)

Connect to your configured servers, enumerate their real tools, and scan them — stdio and remote.

Passive gateway

Sit between an agent and its servers. Log and Ed25519-sign every tool call. Never blocks — zero production risk.

Verify

A tamper-evident, hash-chained record of what your agents actually did — verifiable offline by anyone.

Open core

Free & open source: seeing the risk — scan, live connect, passive logging, signed records, offline verify.
